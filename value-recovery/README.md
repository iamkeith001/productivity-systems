# 價值回收機制 · Value Recovery

很多被忽略的半成品，只要回收得好，仍然能重新創造價值。知識工作最大的浪費，不是沒有努力，而是努力做過的東西沒有被回收，散失在時間裡。

**線上使用：https://iamkeith001.github.io/productivity-systems/value-recovery/**

## 核心概念

半成品不是問題，沒有回收機制才是問題。判斷的重點不是「完成了嗎」，而是「它還能變成什麼」——失敗沒有回收就只是成本，被回收就會變成學習資產。

## 功能

- **回收池**：半成品收件匣——未完成草稿、舊簡報、會議靈感、失敗方案、AI 產出先收進來；用「可回收價值」分類（A 可直接再用／B 可改寫再用／C 可萃取經驗／D 可轉化教學），而不是用完成度分類。
- **再加工**：從池中挑材料，決定新形式（模板／短文／簡報／避錯清單／教學案例／知識卡／SOP），完成後記一筆「原料 → 成果」；統計待回收、已回收與**回收率**。刪除回收紀錄會把材料退回池中。
- **失敗資產**：把失敗事件萃取成一行一行的教訓，存成避錯清單、風險清單、決策紀錄、教學案例或診斷框架。
- **回顧**：回收池、回收成果、回收率、失敗資產（含累計教訓數）＋回收盤點四問。

## 特色

- 純前端單一 HTML 檔案，可離線使用
- 資料只存本機 `localStorage`（key：`value_recovery_v1`）
- 支援 JSON 備份下載／還原
- 深淺色主題（跟隨系統＋手動切換）
- 行動裝置友善（44px 觸控熱區、safe-area）

## 使用方式

直接用瀏覽器開啟 `index.html`，或加到手機主畫面當 App 用。
