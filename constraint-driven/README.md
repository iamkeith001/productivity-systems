# 約束驅動創新 · Constraint-Driven Innovation

把時間、資源與制度限制，轉化為更聚焦、更有創意的解決方案。二戰工程師因材料短缺促成輕量化設計，精實生產與限制理論也指出：當資源不能無限增加，真正重要的是辨識瓶頸、重新設計流程。約束驅動創新不是消極接受限制，而是主動把限制轉化為設計條件。

**線上使用：https://iamkeith001.github.io/productivity-systems/constraint-driven/**

## 核心概念

限制 → 聚焦 → 打破慣性 → 重新組合 → 產生新解法。限制本身不會自動帶來創新，真正產生創新的，是我們如何重新定義限制：不說「只有兩個人，做不到」，改問「在只能使用兩個人的條件下，流程必須如何設計？」。

## 功能

- **課題**：定義真正目標（最終一定要達成的結果，附範本）＋找出核心限制——五種限制類型（時間／預算／人力／制度／技術）點選即顯示對應策略，只鎖定真正影響成果的那一項，不同時處理所有問題。
- **設計**：把限制改寫成設計條件（附句式範本，內建抱怨句 → 設計句對照）；再用五種重新設計手法產生方案——刪除（這步真的需要嗎）／簡化（能否減少欄位審核交接）／合併（兩步能否一次完成）／替代（能否用現有工具）／自動化（重複性工作交給系統）。
- **實驗**：每個設計方案建立最小可行版本——先在哪裡、用多小的規模試＋明確指標，標記試行中／有效／無效；無效也是收穫，回設計換一種手法。
- **回顧**：課題數、設計方案數、試行中、驗證有效；方法摘要含試行後五問（達成目標？新瓶頸？哪些限制可解除？哪些值得保留？可否擴大？）與「限制必須適度」提醒。

## 與同系列思維工具的差別

- **限制條件重構**（`constraint-reframing/`）：心態層——把「限制＝阻礙」翻轉成「限制＝設計條件」，並主動練習加限制（改變你與限制的關係）。
- **約束驅動創新**（本工具）：流程層——六步驟把一個具體課題從目標、核心限制一路做到刪簡合替自的重新設計與最小可行方案驗證（產出可測試的新流程）。
- **創新組合思維**（`innovation-combo/`）：把既有元素重新連結出新價值（生成新方案）。

- 深淺色主題（跟隨系統＋手動切換）
- 行動裝置友善（44px 觸控熱區、safe-area）
