# 第一性原理思維 · First Principles Thinking

拆解問題到最基本的事實，擺脫既有框架，重新建立最有效的解決方案。源自亞里斯多德：要真正理解一件事，就必須找到那些不能再被拆解的基本事實。它最重要的價值是——不接受「一直以來都是這樣做」作為理由。

**線上使用：https://iamkeith001.github.io/productivity-systems/first-principles/**

## 核心概念

類比思考（別人怎麼做我就怎麼做）速度快，但容易複製既有缺陷。第一性原理回到三個問題：真正要達成的結果是什麼？哪些事實一定是真的？如果不受既有做法限制，可以如何重新設計？核心流程：拆解問題 → 找出基本事實 → 移除假設 → 重建方案。

## 功能

- **課題**：定義真正的問題——「如何增加更多抽血櫃檯？」預設了解法，「如何將等待時間降到十分鐘內？」才聚焦真正目標。可同時記下原本的問法（卡片上以刪除線呈現對照）。
- **拆解**：把問題拆成不能再簡化的基本元素（Enter 快速連續新增），逐項標記**事實**（有資料或規則支持，綠邊）或**假設**（只是習慣、推測，琥珀邊）；假設項展開「追問為什麼」欄位——通常追問三到五次，才看見真正原因，那裡就是重新設計的空間。
- **重建**：暫時忘記現有流程——哪些資料可自動取得？哪些步驟只需異常時執行？哪些決策可規則化？每個重建方案附最小可行版本（在哪裡、多小規模驗證＋核心指標），標記試行中／有效／假設被推翻。
- **回顧**：課題、基本元素、待驗假設、驗證有效四統計；方法摘要含適用時機（成本長期過高、問題反覆發生、需要重大創新——日常問題用經驗快速處理，重大問題才回到第一性原理）。

## 與同系列思維工具的差別

- **逆向思維**（`reverse-thinking/`）：反向提問「怎樣一定會失敗」，鎖定高風險因子建防呆（從失敗端逼近）。
- **約束驅動創新**（`constraint-driven/`）：把資源限制改寫成設計條件，用刪簡合替自重組流程（在限制內重設計）。
- **第一性原理思維**（本工具）：拆到不能再拆的基本事實，區分事實與假設，從零重新推導方案（回到地基重蓋）。
