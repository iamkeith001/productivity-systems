# 跨域類比遷移 · Cross-Domain Analogy Transfer

把其他產業已被驗證的成功原理，轉化為自己工作中的解決方案。醫療安全借用航空業的檢核表、軟體開發借用製造業的精實生產、仿生工程從鳥翼與蜂巢獲得靈感——許多創新不是從零發明，而是遷移。重點不是複製表面形式，而是辨認背後真正有效的結構、機制與因果關係。

**線上使用：https://iamkeith001.github.io/productivity-systems/analogy-transfer/**

## 核心概念

類比遷移不是問「別人用了什麼工具？」，而是問「這個工具解決了什麼結構性的問題？」。核心流程：找出相似問題 → 萃取運作原理 → 移除產業外殼 → 重新套用 → 小規模驗證。

## 功能

- **課題**：把問題抽象化——「抽血櫃檯等太久」提升為「需求波動下的有限資源配置問題」（附抽象句式範本），再選問題結構（等候排隊／高風險決策／技能訓練／大量判讀／品質一致／資源配置），提示可借鏡的結構相似領域。
- **轉譯**：核心分頁——四欄轉譯表（原領域元素 → 底層原理 → 我的情境 → 可測試做法），內建八格範例庫（航空檢核表、電商推薦、物流追蹤、遊戲經驗值、金融風險評分、製造看板、餐廳預約分流、版本控制）一鍵帶入原理；每筆轉譯做三重檢核（結構相似／機制有效／條件匹配），3/3 通過才進驗證。
- **驗證**：最小實驗——先選一個流程、少量案例、明確指標，標記試行中／有效／無效；無效也是收穫，回轉譯表調整。
- **回顧**：課題數、轉譯數、通過三重檢核數、遷移成功數。

## 與同系列思維工具的差別

- **創新組合思維**（`innovation-combo/`）：把既有元素重新連結出新價值（生成新方案）。
- **心智模型**（`mental-models/`）：收集思考框架（理解世界的透鏡）。
- **跨域類比遷移**（本工具）：把別的領域已驗證的解法，轉譯到自己的情境（移植解題邏輯）。

## 特色

- 純前端單一 HTML 檔案，可離線使用
- 資料只存本機 `localStorage`（key：`analogy_transfer_v1`）
- 支援 JSON 備份下載／還原
- 深淺色主題（跟隨系統＋手動切換）
- 行動裝置友善（44px 觸控熱區、safe-area）

## 使用方式

直接用瀏覽器開啟 `index.html`，或加到手機主畫面當 App 用。
