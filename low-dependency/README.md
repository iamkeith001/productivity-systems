# 低依賴高彈性 · Low Dependency Design

越少被單點綁住，你的系統就越穩、越自由。概念源自工程界的 Single Point of Failure（單點失效）：很多效率問題不是能力不足，而是工作系統依賴性太高。

**線上使用：https://iamkeith001.github.io/productivity-systems/low-dependency/**

## 核心概念

目標不是完全不依賴任何東西，而是：不被單點綁死、不因局部失效而全面停擺、能快速切換與恢復。傳統效率追求「最快」，高彈性系統追求「可持續」。

## 功能

- **盤點**：單點依賴盤點（工具／平台／特定的人／資料／流程／心理慣性）＋失效衝擊 1–5。**風險分＝衝擊 ×（無備援 3／規劃中 2／就位 1）**，依風險排序，≥9 分標「優先處理」紅框。
- **備援**：替依賴點建替代方案（雙備份／替代工具／SOP 文件化／開放格式／交叉訓練／多元管道）；打勾「已就位」後，對應依賴點的風險分**自動下降**、卡片轉綠。統計備援覆蓋率。
- **心理**：認知依賴改寫（「我一定要有完整時間」→彈性版本）＋彈性演練紀錄（在非理想條件下照樣行動）。
- **回顧**：依賴點、高風險數、備援就位進度、演練次數。

## 特色

- 純前端單一 HTML 檔案，可離線使用
- 資料只存本機 `localStorage`（key：`low_dependency_v1`）
- 支援 JSON 備份下載／還原（工具自己也不做單點依賴）
- 深淺色主題（跟隨系統＋手動切換）
- 行動裝置友善（44px 觸控熱區、safe-area）

## 使用方式

直接用瀏覽器開啟 `index.html`，或加到手機主畫面當 App 用。
