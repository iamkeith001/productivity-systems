# 系統邊界意識 · System Boundary Awareness

你能解決的問題，常取決於你把系統邊界畫在哪裡。概念源自 Bertalanffy 的一般系統理論與控制論：很多問題難解，不是能力不足，而是把問題切得太小。

**線上使用：https://iamkeith001.github.io/productivity-systems/system-boundary/**

## 核心概念

邊界畫得太小，會過度責怪個人、重複修補表面、陷入局部最佳化；邊界畫得更完整，才看得到流程、結構、回饋延遲、資訊斷層與長期副作用——從「處理事件」升級成「理解結構」。

## 功能

- **邊界**：記錄問題＋「小邊界歸因」（現在怪誰）＋擴大邊界後的系統因素（一行一個，渲染成因素標籤）；歸因句以劃線呈現，象徵從責怪模式切換到系統理解模式。
- **瓶頸**：替問題標定真正的限制點（流程／決策權／資訊流／資源配置／環境／生理／結構文化），寫下「系統解法」而不是「再努力一點」。
- **循環**：回饋循環紀錄（正向綠／負向紅、循環鏈、介入點）＋延遲效應觀察（現在的行動→可能的延遲後果→檢查點）。
- **回顧**：問題數、瓶頸數、正／負循環數、延遲觀察數。

## 特色

- 純前端單一 HTML 檔案，可離線使用
- 資料只存本機 `localStorage`（key：`system_boundary_v1`）
- 支援 JSON 備份下載／還原
- 深淺色主題（跟隨系統＋手動切換）
- 行動裝置友善（44px 觸控熱區、safe-area）

## 使用方式

直接用瀏覽器開啟 `index.html`，或加到手機主畫面當 App 用。
