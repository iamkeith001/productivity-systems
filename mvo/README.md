# 最小可行輸出 · Minimum Viable Output

不是追求完美，而是先產出最小但有效的成果，讓學習與迭代開始運作。概念源自精實創業的 MVP（Minimum Viable Product），延伸到知識工作、內容創作、研究與 AI 工作流。

**線上使用：https://iamkeith001.github.io/productivity-systems/mvo/**

## 核心概念

用最少時間、資源與心理阻力，產出第一個可被驗證的成果版本。重點不是粗糙，而是：足以測試、足以使用、足以獲得回饋、足以推動下一輪迭代。「完成第一版」通常比「想像完美版」更重要。

## 功能

- **定義**：替成果定義「最小」與「可行」——核心價值（解決什麼問題、給誰用）、第一版先砍掉什麼、時間盒大小。
- **衝刺**：時間盒倒數計時器（15–120 分鐘），時間限制強迫聚焦與快速決策；時間到照樣出貨，版本自動編號（v1、v2⋯）。
- **回饋**：記錄每一版收到的回饋（自己複盤／使用者／同事／市場／AI）與下一版要改什麼，形成「輸出 → 回饋 → 優化」循環。
- **回顧**：輸出計畫數、累計出貨版本、回饋筆數、迭代最多的計畫與版本軌跡。

## 特色

- 純前端單一 HTML 檔案，可離線使用
- 資料只存本機 `localStorage`（key：`mvo_output_v1`）
- 支援 JSON 備份下載／還原
- 深淺色主題（跟隨系統＋手動切換）
- 行動裝置友善（44px 觸控熱區、safe-area）

## 使用方式

直接用瀏覽器開啟 `index.html`，或加到手機主畫面當 App 用。

## 與「最小可行進展法」的差別

- **最小可行進展**（`mvp-progress/`）：管理「每天推進一小步」的持續習慣。
- **最小可行輸出**（本工具）：管理「先出貨第一版、用回饋驅動迭代」的成果循環。
