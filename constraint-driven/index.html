約束驅動創新

限制不是敵人，
是設計條件。

二戰時的工程師因材料短缺，被迫減少零件、尋找替代材料，反而促成輕量化設計；精實生產與限制理論也都指出：當資源不能無限增加，真正重要的是辨識瓶頸、重新設計流程。限制本身不會自動帶來創新——真正產生創新的，是你如何重新定義限制：限制 → 聚焦 → 打破慣性 → 重新組合 → 產生新解法。

不是把原本的方法做得更辛苦，而是重新設計完成目標的方法。

新課題

定義目標與核心限制

資源充足時，人們習慣加功能、加流程、加人力；限制明確時，才會被迫回答「哪個結果最重要？」。先寫最終一定要達成的結果，再找出真正影響成果的那一項核心限制——不要同時處理所有問題。

還沒有課題從一個「資源不夠、時間不夠」的真實困境開始。

設計條件

選擇課題

語言的改變，會讓思考從抱怨轉向設計： 「我們只有兩個人，做不到。」→「在只能使用兩個人的條件下，流程必須如何設計？」 「沒有預算建立新平台。」→「只能使用現有軟體時，如何先驗證核心功能？」
還沒有課題先到「課題」分頁建立一個。

最小可行方案

小規模驗證

先建立能驗證關鍵假設的最小版本：先用試算表模擬、先在一個單位試行、先處理最常見的案例、先建立單一自動化、或先用人工流程驗證需求——小型實驗可以避免投入大量資源後，才發現方向錯誤。

還沒有可驗證的設計方案先到「設計」分頁建立至少一筆。

方法摘要

六步驟約束創新法

  • 六步驟：定義真正目標 → 找出關鍵限制 → 改寫成設計條件 → 刪除／簡化／合併／替代／自動化 → 最小可行方案 → 根據結果持續優化。
  • 清楚的限制產生三種效果：提高聚焦（確認最重要的成果）、打破慣性（原方法不可行就必須換路）、降低複雜度（減少不必要的功能與決策）。最有價值的創新常常不是增加新功能，而是移除不必要的複雜度。
  • 試行後五問：達成核心目標了嗎？產生新的瓶頸嗎？哪些限制可以解除？哪些反而值得保留？可以擴大到其他場景嗎？
  • 限制必須適度：過度限制會造成品質下降、耗竭與安全風險。約束驅動創新不是永遠「少資源多做事」，而是利用明確邊界，找到更聰明、更簡潔的設計。
