## 核心理念

> 不是問「我還有多少時間？」,而是問「我現在適合做什麼類型的事？」

實踐這套理論靠的是一個回饋迴路:

```
每日打卡能量 → 找出你的能量節律 → 依能量排任務 → 每週抓能量小偷 → 修正
```

## 四大模組

| 分頁 | 對應理論 | 功能 |
|---|---|---|
| **⚡ 打卡** | 做法一 | 早/午/晚花 10 秒,幫 身體 / 專注 / 情緒 / 意義 四種能量打分(1–5);並依當下狀態自動建議「現在適合做什麼」。內建 90/15 專注節奏計時器。 |
| **📊 儀表板** | 做法一 | 「時段 × 星期」能量熱力圖,自動標出你的**黃金時段**;近 14 天趨勢線;總打卡 / 平均 / 連續天數。 |
| **✓ 任務** | 做法二 | 幫任務貼上能量標籤(高能量深度 / 中等 / 低能量雜務),打卡時系統就會把對的任務推給對的狀態。 |
| **🗓️ 檢討** | 做法三+四 | 每週勾選恢復節奏達成度 + 揪出能量小偷,寫下反思;可一鍵複製成純文字貼進筆記。 |

## 怎麼用

### 電腦
直接用瀏覽器打開 `index.html` 即可。

### 開發預覽(本機伺服器)
```bash
python3 -m http.server 4177
# 然後開 http://localhost:4177/index.html
```

### 裝到 iPhone 主畫面(像 App 一樣)
本機檔案無法直接「加到主畫面」成獨立 App,需先讓 iPhone 能連到它,兩個方式擇一:

1. **同網路存取**:Mac 上跑 `python3 -m http.server 4177`,iPhone 開 `http://<Mac的IP>:4177/index.html`,再從 Safari 分享選單選「加入主畫面」。
2. **靜態託管(推薦,隨處可用)**:把 `index.html` 丟到 GitHub Pages / Netlify / Cloudflare Pages,用網址開啟後「加入主畫面」。已內建 App 圖示與全螢幕設定。

## 備份與還原

- 所有資料存在使用者自己的瀏覽器,**不會上傳任何地方**。
- 清除瀏覽器資料 = 紀錄消失,請在「儀表板 → 備份與還原」偶爾按**下載備份**。
- 換裝置:用**讀取備份**還原。

> 介面刻意避開技術名詞(如「匯出 JSON」),改用一般人看得懂的「下載備份 / 讀取備份」。
