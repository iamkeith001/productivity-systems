第一性原理思維

拆到不能再拆，
再重新開始。

亞里斯多德認為，要真正理解一件事，就必須找到那些不能再被拆解的基本事實。類比思考——別人怎麼做我就怎麼做——速度快，但容易複製既有缺陷；第一性原理不接受「一直以來都是這樣做」作為理由，回到三個問題：真正要達成的結果是什麼？哪些事實一定是真的？如果不受既有做法限制，可以如何重新設計？流程：拆解問題 → 找出基本事實 → 移除假設 → 重建方案。

不要只問別人怎麼做，回到事實，重新推導。

新課題

定義真正的問題

不要把現有做法當成問題本身。「如何增加更多抽血櫃檯？」預設了解法；「如何將等待時間降到十分鐘內？」才聚焦真正目標。

還沒有課題挑一個「一直以來都這樣做」的流程開始。

拆解

選擇課題

把問題拆成不能再簡化的基本元素，逐項標記：事實＝有資料或規則支持；假設＝只是習慣、推測或過去經驗。對假設連續追問：為什麼需要這一步？為什麼一定由這個人做？為什麼不能提前？為什麼不能自動化？為什麼不能刪除？——通常追問三到五次，才看見真正原因。
還沒有課題先到「課題」分頁建立一個。

從零重建

如果今天重新建立，我會怎麼設計？

暫時忘記現有流程：哪些資料可以自動取得？哪些步驟只需異常時才執行？哪些決策可以規則化？哪些工作不需要人員介入？先驗證核心假設——試算表模擬、先測一個流程、先用十個案例、先在單一單位試行——避免「重新設計」變成高成本冒險。
還沒有重建方案先在「拆解」把假設攤開，再回來從零設計。

方法摘要

七步驟第一性原理法

  • 七步驟：定義真正的問題 → 拆解成基本元素 → 區分事實與假設 → 連續追問為什麼 → 從零重建方案 → 最小可行版本 → 用結果持續修正。
  • 最常見的障礙不是資訊不足，而是錯誤假設太多：「一定要增加人力」「一定要買新系統」「一定要開會」——這些可能只是習慣，不是事實。
  • 第一性原理比類比思考更耗時間，不是每件小事都要重新發明。適用時機：成本長期過高、問題反覆發生、現有方法已失效、需要重大創新、核心流程有根本性瓶頸。日常問題用經驗快速處理，重大問題回到第一性原理。
  • 它的價值在於區分「必要條件」與「歷史習慣」：很多低效率流程存在，不是因為最合理，而是因為過去一直這樣做。
