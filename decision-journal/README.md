# 決策日誌法 · Decision Journaling

把《決策日誌法（Decision Journaling）》變成日常做決策、防範結果偏誤並進行事後反思的工具。

**立即使用：https://iamkeith001.github.io/productivity-systems/decision-journal/**

單一 HTML 檔、零外部相依、離線可用、資料只存在你自己的裝置（瀏覽器 localStorage）。

---

## 核心理念

> 記錄決策當下的想法，是為了在未來回顧時，不被好壞結果扭曲了當初的判斷品質。

決策日誌是為了解決「結果偏誤（Outcome Bias）」：人們傾向以「結果好壞」去倒推「決策品質」。好結果不等於好決策（可能是運氣好），壞結果也不等於壞決策（可能是運氣差）。

本工具協助你進行完整的決策與複盤流程：

```text
記錄背景情境 → 條列候選方案 → 記錄最終選擇與理由 → 寫下關鍵假設 → 評估信心度與預測結果 → 設定回顧時間 → 複盤實際結果與歸因
```

## 四大分頁

| 分頁 | 功能說明 |
|---|---|
| **日誌** | 查看所有進行中（待複盤/回顧）與已完成複盤的決策卡片。 |
| **記決策** | 結構化記錄決策：主題、候選方案、最終選擇、關鍵假設、信心程度、具體預期與回顧日期。 |
| **複盤** | 實際結果發生後，檢驗當初的假設與預期，評估「過程品質」與「結果好壞」並進行 2x2 歸因。 |
| **統計** | 視覺化呈現「實至名歸」、「運氣不佳」、「僥倖成功」、「理所當然失敗」的分布，以及信心校準度。 |

## 歸因矩陣 (Attribution Matrix)

在複盤時，本系統會依據你對「決策過程」的星等評分與「結果好壞」，自動將決策分類為：

1. **實至名歸 (Deserved Success)**：好的決策過程 + 好的實際結果。
2. **運氣不佳 (Bad Break)**：好的決策過程 + 壞的實際結果（受外部隨機因素影響）。
3. **僥倖成功 (Dumb Luck)**：差的決策過程 + 好的實際結果（完全靠運氣）。
4. **理所當然的失敗 (Deserved Failure)**：差的決策過程 + 壞的實際結果。
