# 逆向思維 · Reverse Thinking

從相反的角度思考問題，找到原本容易被忽略的解決方案。數學家 Jacobi 的「Invert, always invert」被 Charlie Munger 帶進決策世界：與其問「如何成功？」，不如先問「怎樣一定會失敗？」——人們容易忽略錯誤，而避免重大錯誤，本身就是成功的重要來源。

**線上使用：https://iamkeith001.github.io/productivity-systems/reverse-thinking/**

## 核心概念

逆向思維不是唱反調，而是刻意站到相反的位置重新觀察問題。很多真正重要的改善，不是新增更多事情，而是停止做錯的事情。與其追求最佳答案，不如先排除最差答案——最後剩下的，往往就是最好的選擇。

## 功能

- **課題**：定義真正目標（「提高論文產出」太模糊——是投稿成功率還是完成時間？附範本），再把問題完全反過來問（四種反問句式一鍵帶入：怎樣一定會失敗／什麼最浪費／哪些最容易出錯／哪些事一定會毀掉）。
- **失敗**：五分鐘狂列模式——先別評估、全部寫下來（Enter 快速連續新增，附常見失敗因素 chips）；列完再替每項標記「影響大」「容易發生」，兩者都中自動標成紅邊**高風險因子**。
- **防呆**：高風險因子排最前，每項寫改善策略（反過來該做什麼）＋防呆設計（條碼掃描、雙人確認、自動提醒、SOP、Checklist、AI 預警——不要依靠「大家小心一點」），完成標記「防呆已建立」。
- **回顧**：課題數、失敗因素數、高風險因子數、已建立防呆數；方法摘要含 Munger 名言與六步驟。

## 與同系列思維工具的差別

- **情境預演**（`premortem-thinking/`）：針對一個即將啟動的計畫，預想「它已經失敗了」的情境（單一計畫的事前驗屍）。
- **逆向思維**（本工具）：通用的反向提問引擎——任何目標都先反問怎樣必敗，狂列失敗因素、鎖定高風險、建立防呆系統（走向系統設計）。
- **預先決策**（`pre-decision/`）：預先寫好「如果 X 就 Y」的決策規則（自動化未來的選擇）。
